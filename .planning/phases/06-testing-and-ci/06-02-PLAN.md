  <action>
Create `scripts/check-bundle-size.mjs` that:

1. Reads `packages/frontend/.next/build-manifest.json` (must exist from a prior `npm run build`)
2. Identifies initial JS chunks — the files listed under the `pages` key or `rootMainFiles` key that are loaded on every page (framework, main-app, webpack runtime)
3. For each file, reads the actual file from `packages/frontend/.next/static/`, computes its gzipped size using `zlib.gzipSync`
4. Sums gzipped sizes of all initial JS chunks
5. Compares total against 80kB (81920 bytes) budget
6. Prints a summary table showing each file and its gzipped size
7. Exits 0 if under budget, exits 1 with clear error message if over budget

**Important context from Phase 3 decision:** The 80kB budget is known to be exceeded due to `aws-amplify` subscription library (~68KB gzipped alone). The script should still enforce the budget — the CI step will document the current baseline. If the budget is exceeded, the script should print the overage clearly but still be usable (the CI job can be configured to allow-failure or the budget can be adjusted based on the actual baseline).

**Alternative approach:** If parsing `build-manifest.json` proves fragile (Next.js output structure varies), use the simpler approach of summing all `.js` files under `packages/frontend/.next/static/chunks/` that match initial load patterns. Document the chosen approach in a comment at the top of the script.

Add `"check-bundle-size"` script to root `package.json`: `"check-bundle-size": "node scripts/check-bundle-size.mjs"`

Test locally by running `npm run build && npm run check-bundle-size` and verifying the output is reasonable.
</action>
