Create `.github/workflows/ci.yml` with:

**Triggers:**

- `on: pull_request` targeting `main` branch
- `on: push` to `main` branch (catches direct pushes and validates after merge)

**Jobs (4 parallel jobs):**

1. **lint** — `runs-on: ubuntu-latest`
   - `actions/checkout@v4`
   - `actions/setup-node@v4` with `node-version: "20"` and `cache: "npm"` (CICD-02: caches `~/.npm` keyed on package-lock.json)
   - `npm ci`
   - `npm run lint` (runs ESLint on frontend package via root script)

2. **typecheck** — `runs-on: ubuntu-latest`
   - Same checkout + setup-node + npm ci
   - `npm run typecheck` (runs tsc --noEmit on core and functions via root script)
   - Also run `npx tsc --noEmit -p packages/frontend/tsconfig.json` for frontend typecheck (the root typecheck script only covers core and functions)

3. **test** — `runs-on: ubuntu-latest`
   - Same checkout + setup-node + npm ci
   - `npm test` (runs `vitest run` via root script — covers core, functions, frontend projects)

4. **bundle-size** — `runs-on: ubuntu-latest`
   - Same checkout + setup-node + npm ci
   - `.next/cache` restore via `actions/cache@v4` with:
     - `path: packages/frontend/.next/cache`
     - `key: ${{ runner.os }}-next-${{ hashFiles('packages/frontend/**') }}`
     - `restore-keys: ${{ runner.os }}-next-` (CICD-02: caches .next build cache)
   - `npm run build --workspace=packages/frontend`
   - `node scripts/check-bundle-size.mjs`
   - Note: If the 80kB budget is currently exceeded due to aws-amplify (documented Phase 3 constraint), add `continue-on-error: true` to this job with a comment explaining why — prevents blocking all PRs on a known architectural constraint. Remove `continue-on-error` once the budget is met.

**Do NOT:**

- Cache `node_modules` directly (breaks across Node versions — use setup-node `cache: "npm"` instead)
- Use `vitest` without `--run` flag (hangs in watch mode)
- Add branch protection rules (requires the workflow to run on main first — document as a manual follow-up step)

**Job naming:** Use simple names (`lint`, `typecheck`, `test`, `bundle-size`) — these become the required status check names in branch protection settings.
</action>
