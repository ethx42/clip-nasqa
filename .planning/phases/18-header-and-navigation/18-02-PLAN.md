**Strategy for session context in header:** The AppHeader lives in layout.tsx (server component) but session context is only available in session page components (client). Two approaches: (a) render AppHeader in layout without session props, then have session pages render a "header override" portal, or (b) move AppHeader rendering INTO each page component instead of layout.

Use approach (b) — the cleanest: Remove `<AppHeader />` from layout.tsx. Instead, have layout.tsx render just the skip-to-content link, JSON-LD, and `<main>`. Then:

- Landing page (`page.tsx`): renders `<AppHeader />` with no session props at top of its return.
- Join page: renders `<AppHeader />` with no session props.
- Session participant page (`SessionLivePage`): renders `<AppHeader sessionContext={...} />` with session title + live indicator.
- Session host page (`SessionLiveHostPage`): renders `<AppHeader sessionContext={...} shareSlot={...} />` with session title + live indicator + share button.
- Not-found page: renders `<AppHeader />` with no session props.

This keeps the header consistent but allows session-specific content injection without portals or context providers.

**AppHeader updates:**

- The `sessionContext` slot renders between logo/CLIP and the right-side controls. It should contain a truncated session title (`<h1>` with `truncate text-sm font-bold`) and the LiveIndicator. Wrap in a `flex items-center gap-2 min-w-0 flex-1` container so it can shrink.
- Long session titles: The `<h1>` has `truncate` by default. Add an `onClick` handler that toggles a `titleExpanded` state — when expanded, remove `truncate` class and add `whitespace-normal break-words`. Click again to collapse.
- Mobile responsive for session context: On mobile (`sm:hidden` / `hidden sm:block`), hide the "CLIP" text and hide the LiveIndicator text. The session title gets `text-center flex-1` on mobile to center it.
- The `shareSlot` renders to the left of hamburger with `ml-4` spacing from the session context group but no visible divider.

**IdentityEditor** — Update the trigger to show first name:

- Import `useIdentity` hook (already used internally).
- Extract the trigger into a visible section: the `Popover.Trigger` should render `<User className="h-5 w-5" />` icon PLUS, when `name` is set, a `<span className="hidden sm:inline text-sm font-medium text-foreground">{firstName}</span>` next to it. Extract first name: `name?.split(' ')[0]`.
- When anonymous (no name): just show the User icon, no text.
- The trigger styling stays the same (rounded-lg p-2 hover:bg-accent) but add `gap-1.5 flex items-center` to accommodate the name.

**SessionLiveHostPage** — Currently passes `hostToolbar` and `liveIndicator` to SessionShell (which no longer accepts them after plan 01). Instead:

- Render `<AppHeader sessionContext={sessionContextNode} shareSlot={shareSlotNode} />` at the top of the component's return, before `<SessionShell>`.
- `sessionContextNode` = `<><h1 className="truncate text-sm font-bold text-foreground">{session.title}</h1><LiveIndicator connectionStatus={connectionStatus} lastHostActivity={lastHostActivity} /></>`
- `shareSlotNode` = the existing `hostToolbar` prop (which is `<HostToolbar participantUrl={...} />`)
- Remove `hostToolbar` from the component's props interface since it's now passed directly from the host page.tsx as `shareSlot` to the AppHeader rendered here.

Actually, cleaner: keep `hostToolbar` as a prop but rename it to make intent clear, or just pass `participantUrl` directly and render HostToolbar inside SessionLiveHostPage. Check current prop threading from `host/page.tsx` and simplify.

**SessionLivePage** — Same pattern: render `<AppHeader sessionContext={...} />` at top, no shareSlot (participants don't see Share).

**host/page.tsx** — If `hostToolbar` prop is removed from SessionLiveHostPage, update this file to pass `participantUrl` instead and let SessionLiveHostPage handle its own HostToolbar rendering.
