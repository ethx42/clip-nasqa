  <name>Task 1: Expand GraphQL schema to full type set</name>
  <files>infra/schema.graphql</files>
  <action>
Replace the stub contents of infra/schema.graphql with the full schema. Use the exact type definitions from the research file (Pattern 4).

Key decisions (from CONTEXT.md and RESEARCH.md):

- Use tagged union SessionUpdate `{ eventType: SessionEventType!, sessionSlug: String!, payload: AWSJSON! }` — NOT a native GraphQL union. AppSync native union support in subscriptions is inconsistent.
- API key auth — no @aws_api_key directives needed on individual types; the API-level auth covers all fields.
- Subscription uses @aws_subscribe listing "\_stub" as a placeholder mutation — real mutation names added in Phase 3. This ensures the schema compiles in Phase 1 without requiring real mutations.
- Keep Query.\_empty and Mutation.\_stub stubs — they exist only for schema compilation validity.

Include all entity types: Session, Snippet, Question, Reply. Include SessionEventType enum. Include SessionUpdate tagged union. Include Subscription.onSessionUpdate(sessionSlug: String!): SessionUpdate with @aws_subscribe(mutations: ["_stub"]).

The TTL field in AppSync types is AWSTimestamp (Unix epoch integer). The `id` fields are String (ULIDs). Use AWSTimestamp for createdAt — this matches DynamoDB storage format (Unix epoch seconds).

Final schema structure to write:

```graphql
schema {
  query: Query
  mutation: Mutation
  subscription: Subscription
}

type Session {
  slug: String!
  title: String!
  isActive: Boolean!
  createdAt: AWSTimestamp!
  TTL: AWSTimestamp!
}

type Snippet {
  id: String!
  sessionSlug: String!
  type: String!
  content: String!
  language: String
  createdAt: AWSTimestamp!
  TTL: AWSTimestamp!
}

type Question {
  id: String!
  sessionSlug: String!
  text: String!
  fingerprint: String!
  authorName: String
  upvoteCount: Int!
  downvoteCount: Int!
  isHidden: Boolean!
  isFocused: Boolean!
  isBanned: Boolean!
  createdAt: AWSTimestamp!
  TTL: AWSTimestamp!
}

type Reply {
  id: String!
  questionId: String!
  sessionSlug: String!
  text: String!
  isHostReply: Boolean!
  fingerprint: String!
  createdAt: AWSTimestamp!
  TTL: AWSTimestamp!
}

enum SessionEventType {
  SNIPPET_ADDED
  SNIPPET_DELETED
  CLIPBOARD_CLEARED
  QUESTION_ADDED
  QUESTION_UPDATED
  REPLY_ADDED
  PARTICIPANT_BANNED
}

type SessionUpdate {
  eventType: SessionEventType!
  sessionSlug: String!
  payload: AWSJSON!
}

type Query {
  _empty: String
}

type Mutation {
  _stub: String
}

type Subscription {
  onSessionUpdate(sessionSlug: String!): SessionUpdate @aws_subscribe(mutations: ["_stub"])
}
```

**packages/core/src/schemas.ts** — Zod schemas for mutation input validation:
