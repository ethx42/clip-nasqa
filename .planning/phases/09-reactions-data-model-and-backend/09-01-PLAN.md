<task type="auto">
  <name>Task 2: Implement handleReact resolver with atomic toggle, dedup, rate limiting, and ban check</name>
  <files>packages/functions/src/resolvers/reactions.ts</files>
  <action>
Create `packages/functions/src/resolvers/reactions.ts` following the existing resolver pattern from `qa.ts`.

The resolver implements:

1. **Validate emoji** тАФ parse `emoji` argument with `emojiKeySchema.safeParse()`. Throw `INVALID_EMOJI:{emoji}` if invalid.

2. **Ban + rate limit** тАФ call `checkNotBanned(sessionSlug, fingerprint)` then `checkRateLimit(\`reaction#${fingerprint}\`, 30, 60)`. The `reaction#` prefix ensures separate namespace from question rate limits (per user decision).

3. **Toggle-on attempt** тАФ DynamoDB `UpdateCommand` with:
   - `ADD rxn_{emojiKey}_count :one, rxn_{emojiKey}_reactors :fpSet`
   - `ConditionExpression: NOT contains(rxn_{emojiKey}_reactors, :fp) OR attribute_not_exists(rxn_{emojiKey}_reactors)`
   - `:fpSet` is `new Set([fingerprint])` (DynamoDB String Set, NOT array)
   - `ReturnValues: "ALL_NEW"`
   - Key: `PK=SESSION#{sessionSlug}`, `SK=QUESTION#{targetId}` or `REPLY#{targetId}` based on `targetType`

4. **Toggle-off on ConditionalCheckFailedException** тАФ catch CCF, then:
   - `ADD rxn_{emojiKey}_count :negOne DELETE rxn_{emojiKey}_reactors :fpSet`
   - `ConditionExpression: contains(rxn_{emojiKey}_reactors, :fp)`
   - Catch CCF again on toggle-off = idempotent no-op (NOT an error per Pitfall 2 in research)

5. **Build response** тАФ from `ALL_NEW` attributes:
   - `counts`: all 6 emoji counts via `EMOJI_KEYS.map`, use `Math.max(0, ...)` for display safety
   - `reactedByMe`: for each emoji key, check if fingerprint is in the reactors set using `Array.from(reactors ?? []).includes(fingerprint)` (NOT `Set.has()` per Pitfall 5 in research). Override the toggled emoji with the known result (`isNowReacted`).
   - NEVER include the reactors Sets in the response (RXN-09)

6. **Return SessionUpdate** тАФ `eventType: "REACTION_UPDATED"`, `sessionSlug`, `payload: JSON.stringify({ targetId, targetType, emoji, counts, reactedByMe })`

7. **Logging** тАФ use `logger.info` for successful toggle, `logger.warn` is already handled by checkNotBanned/checkRateLimit

Import from:

- `@nasqa/core`: `EMOJI_KEYS`, `emojiKeySchema`, types
- `./index`: `docClient`
- `./logger`: `logger`
- `./rate-limit`: `checkNotBanned`, `checkRateLimit`

Export: `handleReact` as named export.

Use the `tableName()` helper pattern (same as qa.ts). Use `itemSK(targetType, targetId)` helper to build `QUESTION#id` or `REPLY#id`.
</action>
