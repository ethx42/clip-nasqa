**In `packages/functions/src/resolvers/index.ts`:**

1. Add import: `import { handleReact } from "./reactions";`

2. Add case to switch statement (before `default`):

   ```typescript
   case "react":
     result = await handleReact(args);
     break;
   ```

3. Modify the `getSessionData` function to include reaction counts in question and reply mappers:
   - Import `EMOJI_KEYS` from `@nasqa/core`
   - In the questions `.map()`, add a `reactionCounts` field:
     ```typescript
     reactionCounts: JSON.stringify(
       Object.fromEntries(
         EMOJI_KEYS.map((key) => [key, Math.max(0, (item[`rxn_${key}_count`] as number) ?? 0)])
       )
     ),
     ```
   - In the replies `.map()`, add the same `reactionCounts` field with the same pattern
   - Do NOT include `rxn_*_reactors` attributes — these must never leave the Lambda (RXN-09, Pitfall 7 from research)

The `reactionCounts` field is JSON-stringified because the GraphQL type uses `AWSJSON` scalar. The Phase 10 client will `JSON.parse()` it.

Decision (Claude's discretion): Include reaction counts in initial load rather than lazy-fetch. Rationale: the existing getSessionData query returns all SESSION# items in one scan, reaction counts are scalar attributes adding negligible payload size, and lazy-fetching would require a separate round-trip per item.
