  <action>
1. Create `packages/frontend/src/components/session/qr-code.tsx` (Server Component):
   - Import `QRCode` from `qrcode`.
   - Export `async function QRCodeDisplay({ url, size = 200 }: { url: string; size?: number })`.
   - Generate SVG: `const svg = await QRCode.toString(url, { type: 'svg', width: size, margin: 2 })`.
   - Render with `<div dangerouslySetInnerHTML={{ __html: svg }} className="..." />`.
   - Style the container with a white background, rounded corners, and padding so the QR code is always visible regardless of theme.

2. Create `packages/frontend/src/components/session/copy-button.tsx` (`'use client'`):
   - Accept `value: string` and optional `label: string` props.
   - Import `Copy`, `Check` from `lucide-react`.
   - On click: `navigator.clipboard.writeText(value)`, set copied=true state, reset after 2 seconds.
   - Render button with Copy icon normally, Check icon when copied. Use sonner toast `toast.success(t('session.copied'))` if available, or just visual feedback.
   - Import `useTranslations` from `next-intl` for the "Copied!" text.

3. Create `packages/frontend/src/app/[locale]/session/[slug]/success/page.tsx` (Server Component):
   - Receive `params: Promise<{ locale: string; slug: string }>` and `searchParams: Promise<{ raw?: string }>`.
   - Await both params and searchParams. Extract `slug` and `raw` (the one-time raw secret).
   - Call `getSession(slug)` to get session title. If no session, show error / redirect.
   - If `raw` is missing, show a message "This page is only available immediately after session creation."

   Layout (per user decisions — dedicated success page):
   - Session title at top: `<h1>{session.title}</h1>` with a "Session Created!" badge.
   - **Host Secret section**: Display the raw secret in a monospace box with a CopyButton next to it. Large, prominent warning: "Save this secret now — it won't be shown again." (use `t('session.secretWarning')`). Use a yellow/amber warning card style.
   - **Host URL section**: Construct the host URL as `${baseUrl}/session/${slug}/host#secret=${raw}` — display it with a CopyButton. Note: this URL string includes the hash fragment as a static string visible on the page. The key point is that when the host NAVIGATES to this URL, the browser does NOT send the #fragment to the server.
   - **Participant URL section**: Display `${baseUrl}/session/${slug}` with CopyButton.
   - **QR Code section**: Render `<QRCodeDisplay url={participantUrl} />` for participants to scan.
   - A "Go to Host View" link/button pointing to `/session/${slug}/host#secret=${raw}`.

   - Use `process.env.NEXT_PUBLIC_APP_URL ?? 'http://localhost:3000'` for baseUrl construction. In production this would be the deployed URL.
  </action>
