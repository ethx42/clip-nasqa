1. In `packages/frontend/src/actions/session.ts`, add a new Server Action:
   ```
   export async function validateSessionCode(code: string): Promise<"valid" | "invalid" | "ended"> {
     const session = await getSession(code);
     if (!session) return "invalid";
     if (!session.isActive) return "ended";
     return "valid";
   }
   ```
   Import `getSession` from `@/lib/session` (already imported in this file or add it).

2. Add i18n strings to `packages/frontend/messages/en.json` and `packages/frontend/messages/es.json`:
   - `join.tagline`: "Enter the 6-digit session code" / "Ingresa el codigo de sesion de 6 digitos"
   - `join.invalidCode`: "Invalid code. Please try again." / "Codigo invalido. Intenta de nuevo."
   - `join.sessionEnded`: "This session has ended." / "Esta sesion ha terminado."
   - `join.title`: "Join a session" / "Unirse a una sesion"

3. Create `packages/frontend/src/app/[locale]/join/page.tsx` as a `"use client"` page:

   **Layout (per locked decisions — standalone, no nav/footer):**
   - Full-height centered layout (`min-h-dvh flex flex-col items-center justify-center`)
   - Nasqa logo/branding at top — use the clip logo SVG inline (the two-clips-heart) or import from public/images
   - Short tagline below logo (from i18n `join.tagline`)
   - 6 individual digit input boxes in a flex row below the tagline
   - Error message area below inputs

   **OTP Input Implementation:**
   - 6 individual `<input type="text" inputMode="numeric" pattern="[0-9]" maxLength={1}>` elements
   - Each input is a controlled component with state from `digits` array (useState of 6 empty strings)
   - Use an array of 6 refs: `const inputRefs = useRef<(HTMLInputElement | null)[]>([])`
   - `onChange`: if digit entered (0-9), set digit and move focus to `inputRefs.current[index + 1]`
   - `onKeyDown`: if Backspace and current input is empty, move focus to `inputRefs.current[index - 1]`
   - `onPaste` (on first input): intercept paste, extract all digits from pasted text, fill up to 6 boxes. If pasted text contains a URL, extract the last 6-digit sequence (`/(\d{6})/`)
   - Auto-focus first input via `useEffect(() => { inputRefs.current[0]?.focus(); }, [])`
   - Deep-link: read `searchParams.code` — if valid 6-digit string, pre-fill digits and auto-submit

   **Auto-submit on 6th digit:**
   - When all 6 digits are filled (via typing or paste), immediately call `validateSessionCode(digits.join(""))`
   - While validating, disable inputs and show subtle loading state (opacity reduction, not a spinner)
   - On `"valid"`: `router.push(\`/\${locale}/\${code}\`)`
   - On `"invalid"`: trigger shake animation, clear all digits, re-focus first input, show `join.invalidCode` message
   - On `"ended"`: show `join.sessionEnded` message (different from invalid — per locked decision), clear digits, re-focus first input

   **Shake animation:**
   - Wrap the input row in a `<motion.div>` from framer-motion
   - When shake is triggered, animate `x` with keyframes: `[0, -10, 10, -10, 10, 0]` over 400ms
   - Reset shake state after animation completes

   **Styling (per Design System):**
   - Input boxes: `w-12 h-14 sm:w-14 sm:h-16` (44px+ touch target), `text-center text-2xl font-bold`, `rounded-xl`, `border-2 border-border`, `focus:border-indigo-500 focus:ring-2 focus:ring-indigo-500/20`
   - Gap between inputs: `gap-2 sm:gap-3`, with a slightly larger gap (`gap-4`) between digits 3 and 4 (matching the 3+3 grouped display format)
   - Dark mode: use semantic tokens (`bg-background`, `text-foreground`, `border-border`)
   - Error text: `text-sm text-destructive mt-4`
   - Logo area: appropriate spacing above inputs

   **Metadata (SSR portion):**
   - Create a separate metadata export or a server wrapper that provides `generateMetadata` with title "Join a session | nasqa" and description. Since the page is `"use client"`, metadata must come from a parent layout or a `generateMetadata` in the route — add a `layout.tsx` wrapper or export metadata from a separate server file if needed.
   - Alternative: make the page a server component that renders a client `<JoinForm>` component. This allows `generateMetadata` in the same file.
