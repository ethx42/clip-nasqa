**1. Add `graphqlMutation()` to `packages/frontend/src/lib/appsync-client.ts`:**

Keep the existing Amplify configuration and `appsyncClient` export untouched. Add below them:

- Export `AuthConfig` union type: `{ type: 'api-key'; key: string } | { type: 'jwt'; token: string }` (future-proofs for SSO/Cognito per user decision).
- Export `ParticipantMutationName` type: union of `'addQuestion' | 'upvoteQuestion' | 'downvoteQuestion' | 'addReply' | 'react'`. This is the compile-time guard — host mutation names are NOT in this union.
- Export `async function graphqlMutation<T = unknown>(mutationName: ParticipantMutationName, query: string, variables: Record<string, unknown>, auth?: AuthConfig): Promise<T>`.
  - `mutationName` param enforces compile-time type safety — calling with `'pushSnippet'` or any host mutation name is a TypeScript error.
  - Default `auth` to `{ type: 'api-key', key: process.env.NEXT_PUBLIC_APPSYNC_API_KEY! }`.
  - URL from `process.env.NEXT_PUBLIC_APPSYNC_URL!`.
  - Headers: `Content-Type: application/json` + auth header (`x-api-key` for api-key, `Authorization` for jwt).
  - Body: `JSON.stringify({ query, variables })`.
  - Parse response: if `body.errors?.length`, join error messages and throw `new Error(msg)`.
  - If fetch itself throws a TypeError (network failure), re-throw as-is so `safeClientMutation` can classify it.
  - Return `body.data as T`.

**2. Add `safeClientMutation()` to `packages/frontend/src/lib/safe-action.ts`:**

Keep the existing `safeAction` export untouched. Add:

- Export `MutationErrorType` union: `'rate-limit' | 'banned' | 'network' | 'server'`.
- Internal `parseErrorType(err: unknown): MutationErrorType`:
  - If `err` is a TypeError (fetch network failure) OR `navigator.onLine === false` => `'network'`.
  - If error message includes `'RATE_LIMIT_EXCEEDED'` => `'rate-limit'`.
  - If error message includes `'PARTICIPANT_BANNED'` => `'banned'`.
  - Otherwise => `'server'`.
- Export `SafeClientMutationOptions` interface with fields for localized messages: `rateLimitMessage`, `bannedMessage`, `networkMessage`, `serverMessage`, and optional `rateLimitToastId` (defaults to `'rate-limit'`).
- Export `async function safeClientMutation<T>(fn: () => Promise<T>, options: SafeClientMutationOptions): Promise<{ success: true; data: T } | { success: false; error: MutationErrorType }>`:
  - First attempt: call `fn()`.
  - On success: return `{ success: true, data: result }`.
  - On first failure: wait 1 second (`await new Promise(r => setTimeout(r, 1000))`), then retry once (the one-silent-retry rule per user decision).
  - On second failure: classify error via `parseErrorType`.
  - Call `reportError()` (import from `@/lib/report-error`) for logging.
  - Show toast via `toast.error(message, toastOptions)` from sonner:
    - Rate-limit: use `{ id: options.rateLimitToastId ?? 'rate-limit', duration: 4000 }` (prevents stacking per user decision).
    - All other errors: `{ duration: 4000 }`.
  - Return `{ success: false, error: errorType }`.
  - Import `toast` from `sonner` and `reportError` from `@/lib/report-error`.
