**2. Update `actions/session.ts`:**

For `createSession`:

- The function currently calls `const t = await getTranslations("actionErrors")` at the top.
- `t` is used in: (a) validation (`t("titleRequired")`), (b) error fallback (`t("unexpectedError")`), (c) slug failure (`t("slugGenerationFailed")`).
- Move the `getTranslations` call to just before first usage. Since validation happens before the DynamoDB call, the cleanest approach is:
  - Keep `const t = await getTranslations("actionErrors")` at the top ONLY if the title is invalid (it's needed immediately for the validation error). Actually, `getTranslations` is needed before the try block for the `titleRequired` case. So the lazy pattern here is: call `getTranslations` only if `!title` (for the error) or inside the catch/failure paths.
  - APPROACH: Check `!title` first. If invalid, THEN call `getTranslations` and return error. For the DynamoDB loop, if an unexpected error occurs, call `getTranslations` inside the catch. If `!redirectUrl` after loop, call `getTranslations` then.
  - This means `getTranslations` is only called on error paths (empty title, DynamoDB failure, code exhaustion). The happy path (title valid, DynamoDB succeeds, redirect) never calls `getTranslations`.

```typescript
// Pattern for createSession:
export async function createSession(formData: FormData): Promise<ActionResult | never> {
  const raw = (formData.get("title") as string | null)?.trim().slice(0, 50);
  const title = raw?.replace(/\S+/g, (w) => w.charAt(0).toUpperCase() + w.slice(1));
  const locale = await getLocale();

  if (!title) {
    const t = await getTranslations("actionErrors"); // lazy — only on validation error
    return { success: false, error: t("titleRequired") };
  }

  let redirectUrl: string | null = null;

  for (let attempt = 0; attempt < MAX_CODE_RETRIES; attempt++) {
    // ... same loop logic ...
    try {
      // ... PutCommand ...
      redirectUrl = `/${locale}/${code}/host?raw=${rawSecret}`;
      break;
    } catch (err) {
      if (err instanceof ConditionalCheckFailedException) continue;
      const t = await getTranslations("actionErrors"); // lazy — only on unexpected error
      return { success: false, error: t("unexpectedError") };
    }
  }

  if (!redirectUrl) {
    const t = await getTranslations("actionErrors"); // lazy — only on code exhaustion
    return { success: false, error: t("slugGenerationFailed") };
  }

  redirect(redirectUrl);
}
```

```typescript
// BEFORE:
export async function banQuestionAction(args: {...}): Promise<ActionResult> {
  const t = await getTranslations("actionErrors");
  try {
    await appsyncMutation(BAN_QUESTION, args);
    return { success: true };
  } catch (err) {
    reportError(err instanceof Error ? err : new Error(String(err)));
    return parseRateLimitOrBan(err, t, "failedBanQuestion");
  }
}

// AFTER:
export async function banQuestionAction(args: {...}): Promise<ActionResult> {
  try {
    await appsyncMutation(BAN_QUESTION, args);
    return { success: true };
  } catch (err) {

    reportError(err instanceof Error ? err : new Error(String(err)));
    return parseRateLimitOrBan(err, t, "failedBanQuestion");
  }
}
```
