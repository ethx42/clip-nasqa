**2. Create packages/functions/src/resolvers/clipboard.ts:**

Import `docClient` from `./index`, `ulid` from `ulid`, and `PutCommand`, `DeleteCommand`, `QueryCommand`, `BatchWriteCommand` from `@aws-sdk/lib-dynamodb`.

Helper function `tableName()`: reads `process.env.TABLE_NAME` (set in sst.config.ts Lambda environment).

Helper function `verifyHostSecret(sessionSlug, hostSecretHash)`: GetCommand for `PK=SESSION#${slug}, SK=SESSION#${slug}`, compare `.hostSecretHash === hostSecretHash`. Throw `Unauthorized` if mismatch.

Implement three exported async functions:

- `pushSnippet(args)`: verify host secret, generate ULID id, PutCommand with PK/SK/id/sessionSlug/type/content/language/createdAt/TTL, return `{ eventType: 'SNIPPET_ADDED', sessionSlug, payload: JSON.stringify({...snippet fields}) }`

- `deleteSnippet(args)`: verify host secret, DeleteCommand for `PK=SESSION#${slug}, SK=SNIPPET#${snippetId}`, return `{ eventType: 'SNIPPET_DELETED', sessionSlug, payload: JSON.stringify({ snippetId }) }`

- `clearClipboard(args)`: verify host secret, QueryCommand for all `SK begins_with SNIPPET#` items, BatchWriteItem to delete all (handle 25-item batch limit), return `{ eventType: 'CLIPBOARD_CLEARED', sessionSlug, payload: JSON.stringify({}) }`

**3. Create packages/functions/src/resolvers/qa.ts:**

Import same DynamoDB utilities. Implement four exported async functions:

- `addQuestion(args)`: validate text length <= 500, generate ULID id, PutCommand with all Question fields (upvoteCount: 0, downvoteCount: 0, isHidden: false, isFocused: false, isBanned: false, voters: empty Set — use `docClient.createSet ? new Set([])` or initialize without voters attribute and handle in upvote), return QUESTION_ADDED event

- `upvoteQuestion(args)`: UpdateCommand with `ADD upvoteCount :inc` and conditional `contains`/`not contains` on voters set per research Pattern 6. Handle `remove: true` case (decrement). On ConditionalCheckFailedException, throw 'VOTE_CONFLICT'. Return QUESTION_UPDATED with `{ questionId, upvoteDelta }`.

- `addReply(args)`: validate text length <= 500, generate ULID id, PutCommand with Reply fields (PK=SESSION#slug, SK=REPLY#id), return REPLY_ADDED event with full reply payload

- `focusQuestion(args)`: verify host secret. If `questionId` is provided: first Query all QUESTION# items where isFocused=true, UpdateCommand to set isFocused=false on each, then UpdateCommand to set isFocused=true on target question. If `questionId` is null/undefined: unfocus all. Return QUESTION_UPDATED event with `{ questionId, isFocused: true/false }`.

**1. Create packages/frontend/src/lib/appsync-client.ts:**

```typescript
import { Amplify } from "aws-amplify";
import { generateClient } from "aws-amplify/api";

Amplify.configure(
  {
    API: {
      GraphQL: {
        endpoint: process.env.NEXT_PUBLIC_APPSYNC_URL!,
        region: process.env.NEXT_PUBLIC_AWS_REGION ?? "us-east-1",
        defaultAuthMode: "apiKey",
        apiKey: process.env.NEXT_PUBLIC_APPSYNC_API_KEY!,
      },
    },
  },
  { ssr: true },
);

export const appsyncClient = generateClient();
```

This file must NOT have `'use client'` — it is a shared module. Amplify.configure is called at module evaluation time. The `{ ssr: true }` flag enables server-side compatible mode.
