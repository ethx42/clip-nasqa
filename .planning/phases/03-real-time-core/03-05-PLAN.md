**1. Update packages/frontend/src/hooks/use-session-updates.ts:**

Add connection status tracking. Expose `connectionStatus: 'connected' | 'connecting' | 'disconnected'` state.

- Set `connecting` initially and on reconnection attempt.
- Set `connected` when first event arrives or when subscription successfully established.
- Set `disconnected` on error. The Amplify subscription client auto-retries; track state transitions.
- Export connectionStatus alongside the subscription setup.
- Use a ref or state variable for `lastHostActivity: number | null` — update timestamp whenever a SNIPPET_ADDED event arrives.

**2. Create packages/frontend/src/components/session/live-indicator.tsx:**

`'use client'` component. Props: `connectionStatus: 'connected' | 'connecting' | 'disconnected'`, `lastHostActivity?: number | null`.

Render in session header:
- Connected: green pulsing dot (`bg-emerald-500 animate-pulse rounded-full w-2 h-2`) + "LIVE" text in emerald.
- Disconnected: grey dot + "Paused" text in muted.
- Connecting: yellow dot + "Reconnecting..." text.
- If `lastHostActivity` is set and more than 2 minutes ago: show "Last snippet Xm ago" text next to the dot.

**3. Create packages/frontend/src/components/session/new-content-banner.tsx:**

`'use client'` component. Props: `message: string`, `visible: boolean`, `onTap: () => void`.

Render a sticky banner at the top of a panel:
- `position: sticky; top: 0; z-index: 10`
- Emerald background with white text and a subtle shadow.
- Click/tap calls `onTap` (scrolls to top in parent).
- Animate in/out with a slide-down transition.

**4. Update packages/frontend/src/components/session/clipboard-panel.tsx:**

Add host action handlers:
- Delete snippet: each snippet card's three-dot menu has a "Delete" option. On click, call `deleteSnippetAction` (optimistic: dispatch SNIPPET_DELETED immediately).
- Clear All button: near top of clipboard panel (host only). On click, show a confirmation dialog using native `window.confirm("Clear all snippets? This cannot be undone.")` or a simple modal. On confirm, call `clearClipboardAction` and dispatch CLIPBOARD_CLEARED. Show toast via `sonner` toast: "Clipboard cleared by speaker" for 3s (participant side — the subscription event triggers this toast in the event handler).

Add scroll-aware "New snippet" banner:
- Track scroll position using `useRef` on the scroll container.
- When a SNIPPET_ADDED event arrives and user is scrolled down (scrollTop > some threshold), show `<NewContentBanner message="New snippet from speaker" />`.
- Tapping banner scrolls container to top (`scrollTo({ top: 0, behavior: 'smooth' })`).

Add Framer Motion animations to snippet list:
- Wrap snippet items with `motion.div` from framer-motion.
- New snippet (hero): `initial={{ opacity: 0, y: -20 }} animate={{ opacity: 1, y: 0 }}` with emerald highlight border that fades after 2s.
- Deletion: `exit={{ opacity: 0, height: 0 }}` via AnimatePresence.
- Hero-to-history transition: hero slides down using `layout` prop.

Add "Clipboard cleared by speaker" toast trigger: when subscription dispatches CLIPBOARD_CLEARED, call `toast('Clipboard cleared by speaker')` from sonner.
