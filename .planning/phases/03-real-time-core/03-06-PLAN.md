  <how-to-verify>
**Prerequisites:** Deploy the application (`npm run deploy`) or run locally (`cd packages/frontend && npm run dev`). You need two browser windows (or a browser + incognito) to test real-time.

**1. Session Creation and Navigation**
- Create a new session from the landing page
- Copy the host URL (with #secret=...) from the success page
- Open participant URL in a second browser window
- Verify both windows load with the session shell

**2. Clipboard — Host Push (Window 1: Host)**
- Paste a TypeScript code snippet into the host input
- Verify language auto-detection shows "typescript" chip
- Verify live Shiki preview renders below the input
- Press Cmd/Ctrl+Enter or click "Push Snippet"
- Verify snippet appears as Hero in host view
- Switch to participant window — verify snippet appears within ~200ms
- Verify Shiki syntax highlighting renders correctly
- Toggle dark/light theme — verify code colors switch without layout shift

**3. Clipboard — History and Text**
- Push a second code snippet — verify first snippet moves to history (collapsed)
- Push a text snippet (plain text, no code patterns) — verify it shows as "Text" with no syntax highlighting
- Verify snippet numbers (#1, #2, #3) are visible
- Verify copy button works (copies raw content, icon swaps to checkmark for 2s)
- Click a history snippet — verify it expands to show full content

**4. Clipboard — Delete and Clear (Host)**
- Click three-dot menu on a history snippet — click "Delete"
- Verify snippet fades out with animation
- Verify participant sees deletion propagate
- If hero is deleted, verify next snippet promotes to hero
- Click "Clear All" — verify confirmation dialog appears
- Confirm clear — verify all snippets removed
- Verify participant sees "Clipboard cleared by speaker" toast

**5. Q&A — Question Submission (Window 2: Participant)**
- Type a question in the Q&A input
- Verify character counter appears when typing 400+ characters
- Submit the question
- Verify optimistic insert with emerald flash
- Verify "You" badge appears on own question
- Switch to host window — verify question appears

**6. Q&A — Upvoting**
- Click upvote arrow on a question
- Verify arrow turns emerald and count increments immediately
- Verify other window sees the vote change
- Click again to remove vote — verify arrow returns to default

**7. Q&A — Replies**
- Click "Reply" on a question
- Type a reply and submit
- Verify reply appears inline below the question
- Submit a reply from the host window — verify "Speaker" badge and emerald border

**8. Q&A — Focus (Host)**
- Click three-dot menu on a question — click "Focus"
- Verify question pins to top with emerald glow and "FOCUSED" label
- Verify reply thread auto-expands
- Verify participant sees the focused state propagate
- Focus a different question — verify previous unfocuses

**9. Connection and Liveness**
- Verify green pulsing "LIVE" dot in session header
- Verify connected participant count (if implemented)
- (Optional) Kill the dev server briefly — verify yellow "Reconnecting..." banner appears

**10. Mobile Experience**
- Open participant URL on mobile (or Chrome DevTools mobile view)
- Verify tab bar switches between Clipboard and Q&A
- Verify host input is accessible on mobile host view
- Verify all interactions work on mobile

**11. Bundle Size**
- Run `cd packages/frontend && npx next build`
- Check "First Load JS shared by all" is under 80kB gzipped
