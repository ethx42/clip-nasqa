**1. Create packages/frontend/src/hooks/use-fingerprint.ts:**

`'use client'` hook. Exports `useFingerprint()` returning `{ fingerprint: string, votedIds: Set<string>, addVote: (id: string) => void, removeVote: (id: string) => void }`.

- On mount, read `nasqa_fingerprint` from localStorage. If missing, generate a UUIDv4 (`crypto.randomUUID()`) and store it.
- Read `votes:{sessionSlug}` from localStorage as a JSON array of question IDs. Expose as a `Set<string>`.
- `addVote(id)`: add to set, persist to localStorage.
- `removeVote(id)`: remove from set, persist to localStorage.
- Accept `sessionSlug` as parameter: `useFingerprint(sessionSlug: string)`.

**2. Create packages/frontend/src/hooks/use-session-state.ts:**

`'use client'` hook. Exports `useSessionState(initialData: { snippets: Snippet[], questions: Question[], replies: Reply[] })`.

Uses `useReducer` with action types:

```typescript
type SessionAction =
  | { type: 'SNIPPET_ADDED'; payload: Snippet; optimisticId?: string }
  | { type: 'SNIPPET_DELETED'; payload: { snippetId: string } }
  | { type: 'CLIPBOARD_CLEARED' }
  | { type: 'QUESTION_ADDED'; payload: Question; optimisticId?: string }
  | { type: 'QUESTION_UPDATED'; payload: { questionId: string; upvoteDelta?: number; isFocused?: boolean } }
  | { type: 'REPLY_ADDED'; payload: Reply }
  | { type: 'ADD_SNIPPET_OPTIMISTIC'; payload: Snippet }
  | { type: 'ADD_QUESTION_OPTIMISTIC'; payload: Question }
  | { type: 'REMOVE_OPTIMISTIC'; payload: { id: string } }
```

State shape:
```typescript
interface SessionState {
  snippets: Snippet[];       // reverse-chronological (newest first)
  questions: Question[];     // sorted by upvoteCount desc in selector, stored unsorted
  replies: Reply[];          // all replies, grouped by questionId in selector
}
```

Reducer logic:
- `SNIPPET_ADDED`: if `optimisticId`, replace optimistic item by id; otherwise prepend to snippets array. Sort by createdAt desc.
- `SNIPPET_DELETED`: filter out snippet by id.
- `CLIPBOARD_CLEARED`: set snippets to empty array.
- `QUESTION_ADDED`: if `optimisticId`, replace; otherwise append.
- `QUESTION_UPDATED`: find question by id, apply `upvoteDelta` (add to upvoteCount) and/or `isFocused` update. If `isFocused=true`, set all other questions' isFocused to false.
- `REPLY_ADDED`: append to replies array.
- `ADD_SNIPPET_OPTIMISTIC` / `ADD_QUESTION_OPTIMISTIC`: prepend with a temp id (prefix `_opt_`).
- `REMOVE_OPTIMISTIC`: filter out by id (for rollback).

Return `{ state, dispatch }` plus derived helpers:
- `sortedQuestions`: questions sorted with focused first, then upvoteCount desc, then createdAt desc.
- `repliesByQuestion(questionId)`: filter replies for a given question, sorted by createdAt asc.

**3. Create packages/frontend/src/hooks/use-session-updates.ts:**

`'use client'` hook. Exports `useSessionUpdates(sessionSlug: string, dispatch: React.Dispatch<SessionAction>)`.

- Import `appsyncClient` from `@/lib/appsync-client` and `ON_SESSION_UPDATE` from `@/lib/graphql/subscriptions`.
- In `useEffect`: subscribe to AppSync using `appsyncClient.graphql({ query: ON_SESSION_UPDATE, variables: { sessionSlug } })`. The return value is an Observable-like with `.subscribe({ next, error })`.
- On `next`: parse `data.onSessionUpdate`. Switch on `eventType`:
  - `SNIPPET_ADDED`: `JSON.parse(payload)` into Snippet, dispatch `{ type: 'SNIPPET_ADDED', payload: snippet }`.
  - `SNIPPET_DELETED`: parse payload, dispatch `{ type: 'SNIPPET_DELETED', payload }`.
  - `CLIPBOARD_CLEARED`: dispatch `{ type: 'CLIPBOARD_CLEARED' }`.
  - `QUESTION_ADDED`: parse payload into Question, dispatch.
  - `QUESTION_UPDATED`: parse payload (questionId, upvoteDelta, isFocused), dispatch.
  - `REPLY_ADDED`: parse payload into Reply, dispatch.
- On `error`: log to console. Consider setting a connection status state (will be used in Plan 05 for reconnection banner).
- Return cleanup: `sub.unsubscribe()` on unmount.
- Wrap dispatch in `useCallback` to prevent re-subscriptions.

**Deduplication strategy for optimistic + subscription:** When a subscription event arrives for SNIPPET_ADDED or QUESTION_ADDED, check if there's an optimistic item (id starts with `_opt_`) with matching content and close timestamp. If found, replace it. If not found (e.g., event from another client), just prepend. The reducer handles this by checking `optimisticId` in the action.
