**1. Create packages/frontend/src/actions/qa.ts:**

`'use server'` directive. Import `appsyncClient` from `@/lib/appsync-client` and mutation strings from `@/lib/graphql/mutations`.

Four Server Actions:

- `addQuestionAction(args: { sessionSlug: string, text: string, fingerprint: string }): Promise<{ ok: boolean, error?: string }>` — validate text.length <= 500, call ADD_QUESTION mutation via appsyncClient.

- `upvoteQuestionAction(args: { sessionSlug: string, questionId: string, fingerprint: string, remove?: boolean }): Promise<{ ok: boolean, error?: string }>` — call UPVOTE_QUESTION mutation. On VOTE_CONFLICT error, return `{ ok: false, error: 'VOTE_CONFLICT' }`.

- `addReplyAction(args: { sessionSlug: string, questionId: string, text: string, fingerprint: string, isHostReply: boolean }): Promise<{ ok: boolean, error?: string }>` — validate text.length <= 500, call ADD_REPLY mutation.

- `focusQuestionAction(args: { sessionSlug: string, hostSecretHash: string, questionId?: string }): Promise<{ ok: boolean, error?: string }>` — call FOCUS_QUESTION mutation. If questionId is undefined/null, unfocuses all.

**2. Create packages/frontend/src/components/session/question-card.tsx:**

`'use client'` component. Props: `question: Question` (from @nasqa/core), `replies: Reply[]`, `isHost: boolean`, `fingerprint: string`, `sessionSlug: string`, `hostSecretHash?: string`, `votedQuestionIds: Set<string>`, `onUpvote: (questionId: string, remove: boolean) => void`, `onReply: (questionId: string, text: string) => void`, `onFocus?: (questionId: string | undefined) => void`.

Layout per user decisions:
- Left side: upvote arrow (lucide-react `ChevronUp` or `ArrowUp`) + count. Arrow turns emerald when question is in `votedQuestionIds`. Clicking toggles vote via `onUpvote(question.id, isVoted)`.
- Center: question text (auto-linkified URLs using linkify utility from Plan 02). Below text: relative timestamp, reply count badge ("N replies" clickable to expand), "Reply" button.
- Right side: three-dot menu (lucide-react `MoreVertical`) for host actions — visible only when `isHost`. Menu items: "Focus" / "Unfocus" toggle, "Delete" (delete will be wired in Plan 05).
- "You" badge: show a subtle pill badge if `question.fingerprint === fingerprint`.
- Focused state: if `question.isFocused`, wrap the card in a pulsing emerald glow border. Use `tw-animate-css` pulse class or a custom Tailwind animation: `animate-pulse ring-2 ring-emerald-500/50`. Add "FOCUSED" label text.

Reply expansion:
- State: `const [showReplies, setShowReplies] = useState(question.isFocused)` — focused questions auto-expand replies per user decision.
- When expanded: render `<ReplyList>` below the question content.
- When collapsed: show reply count badge. Badge highlights briefly when new replies arrive (Plan 04 will trigger this).

Reply input:
- When "Reply" button is tapped, show an inline text input below the question (above ReplyList).
- 500 char limit with counter at 80%+.
