**detect-language.ts â€” Stricter heuristics (SHIKI-05):**

Rewrite `detectLanguage` with multi-signal confidence requirement. Problems to fix:

- URLs like `https://example.com/path?key=value` match the YAML rule (`/^\s*[\w-]+\s*:/`) because of the `https:` prefix. Fix: check for URL patterns first and return "text".
- Casual text like `hey what's up; let me know()` matches the JS rule (`/[{;]/ && /\w+\s*\(/`). Fix: require at least 2 signals for JS (e.g., semicolons + function calls + curly braces, not just any two).
- Plain text should default to "text" â€” only assign a language when confidence is HIGH (multiple signals).

New heuristic order:

1. Empty/whitespace -> "text"
2. URL detection: if line starts with `http://` or `https://` or matches URL pattern -> "text"
3. JSON: starts with `{` or `[` and has `"key":` pattern -> "json"
4. SQL: starts with SQL keyword (SELECT, INSERT, etc.) -> "sql"
5. HTML: has `<tag>` patterns AND no leading `{` -> "html"
6. Python: `def `, `import `, `from ` with `:` at end of line, or `if __name__`, or `print(` -> "python"
7. Go: `package `, `func `, `:=` -> "go"
8. Rust: `fn `, `let mut`, `impl `, `use std` -> "rust"
9. Java: `public class`, `private `, `System.out` -> "java"
10. C++: `#include`, `int main`, `std::` -> "cpp"
11. TypeScript: requires BOTH (import/export/const/let/function/interface/type keywords) AND (`{` or `}`) AND (`;` or `=>` or `: `) -> "typescript"
12. YAML: starts with `word:` pattern AND no `{` AND no `;` AND line has no URL protocol -> "yaml"
13. JavaScript: requires at LEAST 2 of: (a) curly braces, (b) semicolons at end of statements, (c) `function` or `=>` or `var ` keyword -> "javascript"
14. CSS: has `{` and `:` and `;` pattern with property-value-like structure (e.g., `color:`, `margin:`) -> "css"
15. Bash: starts with `#!/bin/bash` or `#!/bin/sh`, or has `$()` or `${` patterns -> "bash"
16. Default: "text"

Keep the `SUPPORTED_LANGUAGES` array and `LanguageOption` interface unchanged.
