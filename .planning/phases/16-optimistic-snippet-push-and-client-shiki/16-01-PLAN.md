**use-session-state.ts — Extend reducer for failed snippet tracking:**

1. Add two new action types to `SessionAction`:
   - `{ type: "SNIPPET_PUSH_FAILED"; payload: { id: string } }`
   - `{ type: "SNIPPET_RETRY"; payload: { id: string } }`

2. Add `failedSnippetIds: Set<string>` to `SessionState` interface. Initialize as `new Set<string>()` in `useSessionState`.

3. Add reducer cases:
   - `SNIPPET_PUSH_FAILED`: Add `action.payload.id` to `failedSnippetIds` set (create new Set to trigger re-render).
   - `SNIPPET_RETRY`: Remove `action.payload.id` from `failedSnippetIds` set.
   - Update `REMOVE_OPTIMISTIC` to also remove the id from `failedSnippetIds`.

4. Update `SNIPPET_ADDED` case to handle content-fingerprint dedup for subscription-delivered snippets (when no `optimisticId` is passed). When a snippet arrives via subscription without `optimisticId`, check if any existing snippet has an `_opt_` prefixed ID with matching `sessionCode + content + language + type`. If found, replace the optimistic snippet with the real one and remove the `_opt_` ID from `failedSnippetIds`. This mirrors the `QUESTION_ADDED` pattern at lines 89-114.

5. Export `failedSnippetIds` from the hook return value in `SessionStateResult`.

**use-host-snippet-push.ts — New optimistic push hook:**

Create this file following the pattern from `use-session-mutations.ts` (lines 228-270 for question optimistic push). The hook:

1. Accepts `{ sessionCode, hostSecretHash, dispatch, snippetsLength }` params.
2. `handlePush(content: string, lang: string)`:
   - Generate `tempId = "_opt_" + Date.now()`.
   - Determine `type = lang !== "text" ? "code" : "text"`.
   - Build optimistic `Snippet` object with `id: tempId, sessionCode, type, content, language: lang !== "text" ? lang : undefined, createdAt: Math.floor(Date.now() / 1000), TTL: 0`.
   - Dispatch `ADD_SNIPPET_OPTIMISTIC` with the snippet.
   - Call `pushSnippetAction` via `safeAction`. On failure, do one silent retry.
   - If both attempts fail, dispatch `SNIPPET_PUSH_FAILED` with `{ id: tempId }`.
   - Return `{ success: boolean, tempId }` so the caller knows whether to restore input.
3. `handleRetry(tempId: string, content: string, lang: string)`:
   - Dispatch `SNIPPET_RETRY` to clear failed state.
   - Call `pushSnippetAction` via `safeAction`.
   - If fails, dispatch `SNIPPET_PUSH_FAILED` again.
4. `handleDismissFailed(tempId: string)`:
   - Dispatch `REMOVE_OPTIMISTIC` with `{ id: tempId }`.

Import `pushSnippetAction` from `@/actions/snippet`, `safeAction` from `@/lib/safe-action`, and `useTranslations` from `next-intl`. Use `useCallback` for all three handlers.
</action>
