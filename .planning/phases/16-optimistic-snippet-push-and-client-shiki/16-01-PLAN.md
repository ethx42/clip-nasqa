   - `{ type: "SNIPPET_RETRY"; payload: { id: string } }`

   - `{ type: "SNIPPET_EDIT_END"; payload: { id: string; content: string } }`

2. Add `failedSnippetIds: Set<string>` and `editingSnippetIds: Set<string>` to `SessionState` interface. Initialize both as `new Set<string>()` in `useSessionState`.

3. Add reducer cases:
   - `SNIPPET_PUSH_FAILED`: Add `action.payload.id` to `failedSnippetIds` set (create new Set to trigger re-render).
   - `SNIPPET_RETRY`: Remove `action.payload.id` from `failedSnippetIds` set.
   - `SNIPPET_EDIT_START`: Add `action.payload.id` to `editingSnippetIds` set.
   - `SNIPPET_EDIT_END`: Remove `action.payload.id` from `editingSnippetIds` set. Update the snippet's `content` field in `state.snippets` with `action.payload.content`. Also update `type` and `language` if the content changed (re-detect or keep existing).
   - Update `REMOVE_OPTIMISTIC` to also remove the id from `failedSnippetIds` and `editingSnippetIds`.

4. Update `SNIPPET_ADDED` case to handle content-fingerprint dedup for subscription-delivered snippets (when no `optimisticId` is passed). When a snippet arrives via subscription without `optimisticId`, check if any existing snippet has an `_opt_` prefixed ID with matching `sessionCode + content + language + type`. If found, replace the optimistic snippet with the real one and remove the `_opt_` ID from `failedSnippetIds`. This mirrors the `QUESTION_ADDED` pattern at lines 89-114.

5. Export `failedSnippetIds` and `editingSnippetIds` from the hook return value in `SessionStateResult`.

**use-host-snippet-push.ts — New optimistic push hook with deferred push:**

Create this file following the pattern from `use-session-mutations.ts` (lines 228-270 for question optimistic push). The hook:

1. Accepts `{ sessionCode, hostSecretHash, dispatch, snippetsLength, editingSnippetIds }` params. Note: `editingSnippetIds` is passed in so the hook can check if a snippet is being edited.
2. Track pending pushes in a `Map<string, { content: string; lang: string }>` ref (`pendingPushes`). This stores snippets whose server push was deferred because they were being edited.
3. `handlePush(content: string, lang: string)`:
   - Generate `tempId = "_opt_" + Date.now()`.
   - Determine `type = lang !== "text" ? "code" : "text"`.
   - Build optimistic `Snippet` object with `id: tempId, sessionCode, type, content, language: lang !== "text" ? lang : undefined, createdAt: Math.floor(Date.now() / 1000), TTL: 0`.
   - Dispatch `ADD_SNIPPET_OPTIMISTIC` with the snippet.
   - Store in `pendingPushes` ref: `pendingPushes.current.set(tempId, { content, lang })`.
   - Call `executeServerPush(tempId, content, lang)` (internal helper, see below).
   - Return `{ success: boolean, tempId }` so the caller knows whether to restore input.
4. `executeServerPush(tempId: string, content: string, lang: string)`:
   - Internal async helper (not exported).
   - **Deferred push check:** If `editingSnippetIds.has(tempId)`, the snippet is currently being edited. Do NOT push to server yet. Instead, just store/update the pending push in `pendingPushes` ref and return. The push will happen when `handleEditEnd` is called.
   - Otherwise, call `pushSnippetAction` via `safeAction`. On failure, do one silent retry.
   - If both attempts fail, dispatch `SNIPPET_PUSH_FAILED` with `{ id: tempId }`.
5. `handleEditStart(tempId: string)`:
   - Dispatch `SNIPPET_EDIT_START` with `{ id: tempId }`.
   - This signals the hook that any in-flight or pending push for this ID should be deferred.
6. `handleEditEnd(tempId: string, newContent: string, newLang: string)`:
   - Dispatch `SNIPPET_EDIT_END` with `{ id: tempId, content: newContent }`.
   - Update `pendingPushes` ref with the new content: `pendingPushes.current.set(tempId, { content: newContent, lang: newLang })`.
   - Call `executeServerPush(tempId, newContent, newLang)` to send the (possibly edited) content to the server now that editing is done.
7. `handleRetry(tempId: string, content: string, lang: string)`:
   - Dispatch `SNIPPET_RETRY` to clear failed state.
   - Call `pushSnippetAction` via `safeAction`.
   - If fails, dispatch `SNIPPET_PUSH_FAILED` again.
8. `handleDismissFailed(tempId: string)`:
   - Dispatch `REMOVE_OPTIMISTIC` with `{ id: tempId }`.
   - Remove from `pendingPushes` ref.

Import `pushSnippetAction` from `@/actions/snippet`, `safeAction` from `@/lib/safe-action`, and `useTranslations` from `next-intl`. Use `useCallback` for all exported handlers. Use `useRef` for `pendingPushes`.
