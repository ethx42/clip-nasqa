    2. Create `packages/functions/src/resolvers/logger.ts`:
       - Import pino and create a module-level singleton: `export const logger = pino({ level: process.env.LOG_LEVEL ?? 'info' })`
       - Module-level initialization is critical for Lambda warm starts (avoids re-init overhead)

    3. Modify `packages/functions/src/resolvers/index.ts` handler:
       - Import `logger` from `./logger`
       - At the start of the handler, create a child logger: `const log = logger.child({ operation: fieldName, sessionSlug: args?.sessionSlug ?? 'unknown' })`
       - Record `const start = Date.now()` before the switch
       - Wrap the switch result: on success, `log.info({ durationMs: Date.now() - start }, 'resolver success')` then return result
       - On error catch: `log.error({ durationMs: Date.now() - start, err }, 'resolver error')` then re-throw
       - Log level strategy (Claude's discretion): INFO for successful completions, ERROR for unexpected throws

    4. Modify `packages/functions/src/resolvers/rate-limit.ts`:
       - Import `logger` from `./logger`
       - In `checkRateLimit`, when throwing RATE_LIMIT_EXCEEDED:
         - Calculate remaining seconds: `const remaining = bucket + windowSeconds - Math.floor(Date.now() / 1000)`
         - Encode in error message: `throw new Error(\`RATE_LIMIT_EXCEEDED:\${remaining}\`)`
         - Log at WARN level: `logger.warn({ key, remaining }, 'rate limit exceeded')`
       - In `checkNotBanned`, when throwing PARTICIPANT_BANNED:
         - Log at WARN level: `logger.warn({ sessionSlug, fingerprint: fingerprint.slice(0, 8) }, 'banned participant attempted action')`
         - Only log first 8 chars of fingerprint (privacy, per research: no full fingerprint in logs)
