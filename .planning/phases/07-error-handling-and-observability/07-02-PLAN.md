       ```
       }
       ```
       Provide proper Spanish and Portuguese translations for es.json and pt.json.

    3. Refactor `packages/frontend/src/actions/session.ts`:
       - Import `ActionResult` from `@/lib/action-result`
       - Import `getTranslations` from `next-intl/server`
       - Change return type from `Promise<never>` to `Promise<ActionResult | never>` (success still redirects, which returns never)
       - At the start, get translations: `const t = await getTranslations('actionErrors')`
       - Replace `throw new Error("Session title is required")` with `return { success: false, error: t('titleRequired') }`
       - Replace `throw new Error("Failed to generate unique slug...")` with `return { success: false, error: t('slugGenerationFailed') }`
       - Keep the `redirect()` call outside try/catch (it throws NEXT_REDIRECT)
       - Wrap the unexpected DynamoDB error re-throw in a return: `return { success: false, error: t('unexpectedError') }`
       - CRITICAL: `redirect()` must remain outside try/catch blocks

    4. Refactor `packages/frontend/src/actions/qa.ts`:
       - Import `ActionResult` from `@/lib/action-result`
       - Import `getTranslations` from `next-intl/server`
       - Change all return types from `Promise<{ ok: boolean; error?: string }>` to `Promise<ActionResult>`
       - Replace `{ ok: true }` with `{ success: true }` and `{ ok: false, error: "..." }` with `{ success: false, error: t('key') }`
       - In catch blocks, parse error message for `RATE_LIMIT_EXCEEDED:(\d+)` pattern:
         - If match: extract seconds, return `{ success: false, error: t('rateLimited', { seconds }) }`
         - If `PARTICIPANT_BANNED`: return `{ success: false, error: t('banned') }`
         - Otherwise: return `{ success: false, error: t('failedXxx') }` with appropriate key
       - For `upvoteQuestionAction`: keep `VOTE_CONFLICT` as a special string return (not localized, it's a control code consumed by the client, not shown to users)
       - Replace `console.error` calls with `reportError` from `@/lib/report-error` (created in plan 01)

    5. Refactor `packages/frontend/src/actions/moderation.ts`:
       - Same pattern: import ActionResult, getTranslations
       - Change all returns from `ok` to `success`
       - Add i18n error messages using `actionErrors` namespace
       - Parse RATE_LIMIT_EXCEEDED and PARTICIPANT_BANNED from catch blocks
       - Replace `console.error` with `reportError`

    6. Refactor `packages/frontend/src/actions/snippet.ts`:

       - Replace `console.error` with `reportError`
