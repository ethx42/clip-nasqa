**Create `packages/frontend/src/hooks/use-reaction-state.ts`:**

A "use client" hook managing optimistic reaction state for a single item (question or reply).

Props interface:

```typescript
interface UseReactionStateOptions {
  sessionSlug: string;
  targetId: string;
  targetType: "QUESTION" | "REPLY";
  serverCounts: string | undefined; // raw AWSJSON from state (question.reactionCounts)
  fingerprint: string;
}
```

Return: `{ displayCounts: ReactionCounts; activeEmojis: Set<EmojiKey>; toggle: (emoji: EmojiKey) => void }`

Implementation details:

1. **parseReactionCounts helper** (module-level, exported for reuse): Takes `raw: string | undefined | null`, returns `ReactionCounts`. Uses `EMOJI_KEYS` from `@nasqa/core` to build a zero-filled default object. Merges parsed JSON over defaults with try/catch fallback.

2. **localStorage persistence**: Key format `reactions:{sessionSlug}:{targetId}`. On mount, read and parse into `Set<EmojiKey>`. On toggle, persist updated set. Use try/catch around all localStorage calls (SSR safety).

3. **Optimistic counts**: Maintain `localOverrideCounts: ReactionCounts | null` state (initialized `null`). When user toggles:
   - Flip the emoji in `activeEmojis` set
   - Compute new counts: take current `displayCounts`, increment or decrement the toggled emoji by 1 (floor at 0)
   - Set `localOverrideCounts` to the computed counts
   - Persist `activeEmojis` to localStorage

4. **300ms debounce**: Use `useRef<ReturnType<typeof setTimeout>>` for the debounce timer. On each toggle call, `clearTimeout` the existing timer and set a new 300ms timeout that:
   - Calls `reactAction({ sessionSlug, targetId, targetType, emoji, fingerprint })`
   - On `success: false`: silently revert — restore previous `activeEmojis` and clear `localOverrideCounts` (NO toast per CONTEXT.md)
   - On success: clear `localOverrideCounts` (subscription event will provide authoritative counts)

5. **Server count sync**: When `serverCounts` prop changes (detected via `useEffect` dependency), clear `localOverrideCounts` to `null`. The parsed `serverCounts` becomes the display source. This ensures subscription events always win.

6. **displayCounts**: `localOverrideCounts ?? parseReactionCounts(serverCounts)`

7. **Cleanup**: `useEffect` cleanup clears the debounce timeout on unmount.

Import `reactAction` from `@/actions/reactions`.

**Create `packages/frontend/src/components/session/reaction-bar.tsx`:**

A "use client" component rendering the Slack-style reaction bar.

Props:

```typescript
interface ReactionBarProps {
  sessionSlug: string;
  targetId: string;
  targetType: "QUESTION" | "REPLY";
  reactionCounts: string | undefined; // raw AWSJSON from question/reply
  fingerprint: string;
  className?: string;
}
```

Implementation details:

1. Call `useReactionState` with the props to get `displayCounts`, `activeEmojis`, `toggle`.
2. Use `useTranslations("reactions")` for ARIA labels.
3. Import `EMOJI_PALETTE` from `@nasqa/core` for emoji character lookup and iteration order.

**Render structure:**

- Outer `<div>` with `className={cn("flex flex-wrap items-center gap-1.5", className)}`
- **Active reaction pills**: Filter `EMOJI_PALETTE` to entries where `displayCounts[key] > 0`. For each, render a `<button>`:
  - `aria-label`: Use `t(activeEmojis.has(key) ? "emojiLabelActive" : "emojiLabel", { emoji: t(key), count: displayCounts[key] })` for proper i18n
  - `aria-pressed={activeEmojis.has(key)}`
  - `onClick={() => toggle(key)}`
  - Classes: `cn("inline-flex items-center gap-1 rounded-full px-2.5 py-1 text-sm min-h-11 min-w-11 transition-colors select-none", activeEmojis.has(key) ? "bg-muted text-foreground" : "bg-transparent text-muted-foreground hover:bg-muted/40")`
  - Content: `<span aria-hidden="true">{emoji character}</span>` + `<span className="tabular-nums text-xs font-semibold">{formatCount(count)}</span>`
  - `formatCount(n)`: returns `n > 99 ? "99+" : String(n)`

- **Add-reaction trigger button**: Always rendered (even when pills exist). SmilePlus icon from lucide-react (or Smile icon — pick whichever better mirrors Slack's affordance).
  - `aria-label={t("addReaction")}`
  - `aria-expanded={pickerOpen}` (track with `useState<boolean>(false)`)
  - `onClick` toggles `pickerOpen`
  - Classes: `min-h-11 min-w-11 rounded-full p-2 text-muted-foreground hover:bg-muted/40 transition-colors`

- **Inline emoji picker**: Conditionally rendered when `pickerOpen === true`. NOT a popover — render inline in the flex flow.
  - Render all 6 emojis from `EMOJI_PALETTE` as buttons
  - Each button: `onClick` calls `toggle(key)` then `setPickerOpen(false)` (auto-close after selection per CONTEXT.md)
  - `aria-label={t(key)}` for each picker button
  - Classes: same 44px touch target minimum (`min-h-11 min-w-11 rounded-full`)
  - Use a subtle entrance: can use framer-motion `motion.div` with `initial={{ opacity: 0, width: 0 }}` and `animate={{ opacity: 1, width: "auto" }}` for inline expansion, OR simple CSS transition. At Claude's discretion per CONTEXT.md. Keep it subtle and calm.

**Anti-patterns to avoid (per CONTEXT.md and RESEARCH.md):**

- Do NOT use brand indigo for active highlight — use `bg-muted` (neutral gray)
- Do NOT animate count number changes — instant swap only
- Do NOT use a floating popover for the emoji picker — inline expansion
- Do NOT show toast on reaction error — silent revert
- Do NOT scale buttons by count — uniform size
- Do NOT hardcode emoji characters — always use `EMOJI_PALETTE` from `@nasqa/core`
- Fixed button width regardless of count — use `tabular-nums` and consistent padding to prevent layout shifts
