1. In `packages/frontend/src/hooks/use-session-state.ts`:
   - Import `ReactionCounts` from `@nasqa/core` (type-only import)
   - Add a new variant to the `SessionAction` union:
     ```typescript
     | {
         type: "REACTION_UPDATED";
         payload: {
           targetId: string;
           targetType: "QUESTION" | "REPLY";
           counts: ReactionCounts;
         };
       }
     ```
   - Add the `REACTION_UPDATED` case to `sessionReducer`. It should:
     - Extract `targetId`, `targetType`, `counts` from `action.payload`
     - `JSON.stringify(counts)` to produce the AWSJSON string
     - If `targetType === "QUESTION"`: map over `state.questions`, for the matching question set `reactionCounts` to the stringified counts
     - If `targetType === "REPLY"`: map over `state.replies`, for the matching reply set `reactionCounts` to the stringified counts
     - Return updated state

2. In `packages/frontend/src/hooks/use-session-updates.ts`:
   - Import `ReactionCounts` from `@nasqa/core` (type-only import)
   - Add a `case "REACTION_UPDATED"` in the subscription event switch, BEFORE the `default` case:
     ```typescript
     case "REACTION_UPDATED": {
       const { targetId, targetType, counts } = parsed as {
         targetId: string;
         targetType: "QUESTION" | "REPLY";
         counts: ReactionCounts;
       };
       stableDispatch({
         type: "REACTION_UPDATED",
         payload: { targetId, targetType, counts },
       });
       break;
     }
     ```
   - IMPORTANT: Do NOT dispatch `reactedByMe` from the parsed payload. Each client tracks its own active state locally. Dispatching `reactedByMe` from other clients' events would corrupt the current user's highlights.
