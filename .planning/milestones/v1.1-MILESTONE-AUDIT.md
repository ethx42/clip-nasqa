## Cross-Phase Integration

### Verified Integration Points (17/20)

All critical cross-phase wiring is intact:

- Phase 5 pre-commit hook invokes lint-staged + typecheck
- Phase 6 CI pipeline runs lint, typecheck, test, bundle-size in parallel
- Phase 7 `ActionResult` type imported by all 4 action files; `safeAction` used at 15 call sites
- Phase 7 error boundaries are framework-consumed (Next.js file-based routing)
- Phase 8 SEO files are framework-consumed; `generateMetadata` calls `getSession` + `getSessionData`
- Phase 8 skip-to-content `#qa-feed` anchor targets QAPanel's `id="qa-feed"`

### Integration Gaps (3 items — all low severity)

**INT-01: Pre-commit hook skips frontend typecheck**

- `.husky/pre-commit` runs `npm run typecheck` which covers core + functions only
- CI adds `npx tsc --noEmit -p packages/frontend/tsconfig.json` as a separate step
- Frontend type errors escape locally, caught only in CI
- Fix: add one line to `.husky/pre-commit`

**INT-02: No tests for Phase 7 error handling code**

- `safeAction`, `ActionResult`, `global-error.tsx`, `error.tsx`, `not-found.tsx` have zero test coverage
- These are framework-consumed files difficult to unit test, but `safeAction` and `ActionResult` are testable

**INT-03: Rate-limit test coverage gap**

- 8 existing rate-limit tests predate Phase 7 modifications
- `RATE_LIMIT_EXCEEDED:{remaining}` throw format and pino `logger.warn` are untested
- Client regex parsing depends on this format — a format change would silently break cooldown display
