## Directory Purposes

**packages/core:**

- Purpose: Shared code consumed by frontend and backend packages
- Contains: TypeScript type definitions, Zod schemas, i18n keys
- Key files: `src/index.ts`, `src/types.ts`, `src/schemas.ts`, `src/i18n.ts`

**packages/frontend:**

- Purpose: User-facing web application built with Next.js
- Contains: React components, pages, layouts, CSS, static assets
- Key files: `src/app/layout.tsx`, `src/app/page.tsx`, `next.config.ts`

**packages/functions:**

- Purpose: Backend Lambda functions implementing GraphQL resolvers
- Contains: Serverless function handlers, resolver logic
- Key files: `src/index.ts`, `src/resolvers/`

**infra:**

- Purpose: Infrastructure-as-Code and API schema definitions
- Contains: GraphQL schema, SST configuration
- Key files: `schema.graphql`

**.planning/codebase:**

- Purpose: GSD codebase analysis and planning documents
- Contains: Architecture, structure, conventions, testing patterns, concerns documentation
- Generated by: GSD mapping agents

## Key File Locations

**Entry Points:**

- `packages/frontend/src/app/layout.tsx`: Root React layout wrapping entire Next.js application
- `packages/frontend/src/app/page.tsx`: Home page component mounted at `/`
- `packages/functions/src/index.ts`: Exports all Lambda resolver handlers
- `sst.config.ts`: Infrastructure configuration entry point for deployment

**Configuration:**

- `tsconfig.json`: Root TypeScript configuration extended by all packages
- `packages/frontend/tsconfig.json`: Frontend-specific TypeScript config with path alias `@/*`
- `packages/core/tsconfig.json`: Core package TypeScript config
- `packages/functions/tsconfig.json`: Functions package TypeScript config
- `packages/frontend/next.config.ts`: Next.js configuration
- `sst.config.ts`: SST infrastructure configuration

**Core Logic:**

- `packages/core/src/types.ts`: AppSyncContext interface and other shared types
- `packages/core/src/schemas.ts`: Zod validation schemas (baseSchema)
- `packages/core/src/i18n.ts`: i18n translation key constants
- `packages/functions/src/resolvers/`: GraphQL resolver implementations
- `packages/frontend/src/lib/utils.ts`: Shared utility functions (cn for classname merging)

**Testing:**

- No test files detected in current structure

**Styling:**

- `packages/frontend/src/app/globals.css`: Global Tailwind CSS

## Naming Conventions

**Files:**

- TypeScript: `.ts` for non-React files, `.tsx` for React components
- Resolvers: Named after operation (e.g., `example.ts`), handlers exported with `handler` name
- Components: PascalCase (e.g., `button.tsx` exports `Button` component)
- Utilities: camelCase (e.g., `utils.ts`)

**Directories:**

- Functional grouping: `components/`, `resolvers/`, `lib/`, `app/` (Next.js convention)
- UI components co-located: `components/ui/`
- Package structure: lowercase with hyphens in package names (`@nasqa/core`)

**Exports:**

- Barrel files: `index.ts` in packages re-exports from submodules
- Shared code: Exported from `@nasqa/core` using TypeScript path alias
- Resolvers: Named handler exports re-exported from package index

## Where to Add New Code

**New Feature:**

- Primary code: Add resolver to `packages/functions/src/resolvers/[feature-name].ts`
- Schema: Add type to `packages/core/src/types.ts` and schema to `packages/core/src/schemas.ts`
- GraphQL: Add Query/Mutation/Subscription to `infra/schema.graphql`
- Frontend: Add page/component to `packages/frontend/src/app/` or `packages/frontend/src/components/`
- Tests: Create `[feature].test.ts` co-located with source file

**New Component/Module:**

- UI Component: `packages/frontend/src/components/[component-name].tsx`
- Utility function: `packages/frontend/src/lib/[utility-name].ts`
- Shared type: `packages/core/src/[category].ts` (types, schemas, etc)

**Utilities:**

- Shared helpers: `packages/frontend/src/lib/` for frontend utilities
- Shared across packages: `packages/core/src/` for cross-package utilities
- Style utilities: `packages/frontend/src/lib/utils.ts` (currently contains `cn` function)

## Path Aliases

**Root tsconfig.json:**

- `@nasqa/core`: Maps to `packages/core/src`
- `@nasqa/core/*`: Maps to `packages/core/src/*`

**Frontend tsconfig.json:**

- `@/*`: Maps to `packages/frontend/src/*`
