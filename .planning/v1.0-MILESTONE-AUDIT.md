## Integration Check

### Critical Issues

**1. Hard delete subscription events not handled (P0)**

`use-session-updates.ts` subscription switch does not handle `QUESTION_HARD_DELETED`, `REPLY_HARD_DELETED`, or `SNIPPET_HARD_DELETED`. Lambda resolvers emit these events correctly, and the reducer in `use-session-state.ts` has all three cases — but the subscription bridge drops them silently. Other connected clients never see hard deletes in real-time.

**Fix:** Add three cases to `use-session-updates.ts` switch block.

**2. Hard delete snippet has no UI trigger (P1)**

`handleHardDeleteSnippet` is extracted from `useHostMutations` in `session-live-host-page.tsx` but never passed to `ClipboardPanel`. No prop on `ClipboardPanel`, no menu item on `SnippetCard`. Dead handler.

**Fix:** Thread `onHardDeleteSnippet` prop through `ClipboardPanel` → `SnippetCard`, add "Permanently Delete" overflow menu item with confirmation dialog.

### Minor Issues

- `linkify.tsx` now single-consumer (only `shiki-block.tsx`) — orphan-adjacent
- Hard delete optimistic dispatch uses soft-delete action types — functionally equivalent but semantic mismatch

### Broken Flows

1. **Hard delete → real-time propagation**: breaks at `use-session-updates.ts` (event not handled)
2. **Host hard-deletes snippet**: breaks at UI (no trigger, orphaned handler)
